Security Lab Builder

Host OS: Windows/Linux
Hypervisor: VMware Pro

Lab Overview & System Prerequisites

Welcome to your cybersecurity home lab builder. This section outlines the purpose of the lab and ensures your host machine is ready. You will build a comprehensive environment using VMware Workstation Pro to safely practice penetration testing and SIEM detection.

⚠️ Hardware Detection Profile Detected: 16 GB RAM & 512 GB SSD

Because you are running on 16 GB of RAM, you must optimize allocations so that you do not overcommit your system memory or lock your host OS.

🖥️ Your System Limits Analysis

  • ☑️
    Physical RAM: 16 GB We must operate with extreme efficiency. Windows 11 host uses ~4.5 GB. pfSense is very light (1 GB). We will scale Kali down to 2-3 GB and run other systems only when actively debugging them.
  • ⚠️
    Storage Budget: 512 GB SSD • Windows 11 host OS system files: ~60 GB
    • Your Pre-existing Kali VM: 100 GB
    • Your Pre-existing Windows 11 VM: 50 GB
    • Remaining SSD Space: ~300 GB maximum. Use "Thin Provisioning" during virtual disk creations so that VMs only use the actual disk space they currently require on your drive.

BIOS Configuration Check

Ensure hardware virtualization is enabled in your BIOS before deploying the pfSense Firewall, otherwise VMs will throw a VT-x execution error.

  1. Reboot your physical laptop and enter your BIOS setup (often F2, F10, or ESC).
  2. Navigate to Security, System Options, or Advanced.
  3. Enable Virtualization Technology (VTx) and VTd (Intel) or SVM Mode (AMD).
  4. Save changes and boot back into Windows.

Lean RAM Allocation Strategy (16GB Host)

Optimized so you can run the core components without freezing your host laptop. Turn off components when not in use.

pfSense Firewall

The central brain of your lab. It acts as a NAT firewall for the entire environment, separating your vulnerable VMs from your host's real network. You will configure specific firewall rules here to control traffic between subnets.

Key Properties
  • • Provides granular control over VMware's default NAT.
  • • Routes traffic between all defined lab subnets.
  • • Protects host network from lab malware/attacks.

💡 Crucial Concepts to Know Before Starting pfSense Configuration

Setting up pfSense requires setting up **Virtual Switches** (VMnets) within VMware's Virtual Network Editor. Here is how pfSense bridges them:

1. Interface #1: WAN (VMnet8 NAT)

Connects the firewall directly to VMware's default NAT network. This allows pfSense to retrieve outbound Internet access and download its configurations/packages.

2. Interface #2: LAN (VMnet2 Host-Only)

Connects to your custom, isolated LAN. Your Kali VM will also be assigned to this exact VMnet2 switch so they can communicate together securely.

3. The Goal of pfSense

By disabling direct VMware NAT for targets, Kali and targets cannot talk directly to your physical router or home network. They *must* talk via pfSense.
